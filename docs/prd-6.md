# 补充需求文档 (PRD-6) - 视觉体验与 UI 交互精调

本文档（PRD-6）作为前期需求的补充，基于树状结构原型的初步效果，针对节点图标、高亮逻辑、底色冲突以及底部状态栏溢出等 TUI 细节痛点进行了具体的体验调优规范。

## 1. 树状节点视觉区分与高亮机制 (Node Styling & Highlight)

为了避免界面产生厚重的“斑马线”错觉并维持 `oxker` 风格的极简、克制的美学，特规范以下 UI 渲染准则：

### 1.1 静态层级区分 (Non-focused State)
* **Group 节点 (文件夹/时间组)**：
  * **禁止使用常驻底色。**
  * 使用加粗 (Bold) 配合主题色 (如 Cyan 或 Blue) 作为前景色。
  * 根据当前分组模式增加专属图标前缀：项目模式推荐使用 `📂`，时间模式推荐使用 `🕒`（例如：`📂 openagents (1)`）。
* **Session 节点 (叶子节点)**：
  * 保持标准灰白色前景色。
  * 前缀保持极简的对齐符号（如 `·` 或 `└─`），并严格控制缩进对齐。

### 1.2 绝对焦点指示 (Single Focus Highlight)
* 无论是 Group 节点还是 Session 节点，一旦获取到用户的操作游标 (Focused)，必须且只能由它应用**全行底色反转高亮 (Reverse / Solid Background)**。
* 结果达成：通过“图标+前景色”区分数据层级，通过唯一的“高亮底色块”指示当前操作目标，从设计上杜绝底色混用引发的视觉冲突。

### 1.3 移除冗余的指示符 (Remove Redundant Indicators)
* **痛点**：此前的版本中，高亮行首附带了 `>>` 符号，不仅占用字符宽度，还会打破子节点原本对齐好的垂直视觉线。
* **要求**：既然底色反转已经提供了足够强烈的选中暗示，要求将组件（如 `tui-tree-widget` 的 `highlight_symbol`）配置为空字符串 `""` 或相应数量的空格，彻底移除 `>>` 符号。

## 2. 底部提示栏精简与重构 (Bottom Bar Optimization)

针对当前状态栏过长（超 150 字符），在标准终端小窗（80-100 列）必定发生截断的问题，采取“高频常驻 + 低频收纳”的设计模式：

### 2.1 常驻栏的极客化精简 (Minimal Status Bar)
底部常驻栏仅保留最高频的核心导航操作，移除长单词，采用标准终端缩写。
* **新版文案范例**：`Tab:Engine | j/k:Nav | Enter:Toggle/Resume | d:Del | ?:Help | q:Quit`。
* 确保默认状态下的说明文字长度远低于常规屏幕宽度底线。

### 2.2 引入上下文帮助浮层 (Help Modal)
* 新增全局快捷键映射：`?` （或 `h`）。
* **交互表现**：按下该键后，在终端正中央唤起一个置顶的浮层面板 (Popup Modal)。
* **承载内容**：完整展示所有未在底部常驻的高阶快捷键（如：`Ctrl+Alt+H/V : 切换面板布局`、`Ctrl+Alt+←/→ : 切换面板焦点`、`Ctrl+Alt+=/- : 动态调整大小` 等）。用户按任意键或 `Esc` 键退出该帮助视图。
