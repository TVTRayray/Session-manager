# 补充需求文档 (PRD-2)

此文档作为主需求文档 (`prd.md`) 的延伸，主要提供第二阶段界面的优化和会话连通性的具体要求。

## 1. 交互界面扩展 (UI Enhancements)

### 1.1 全局标题栏 (Global Title)
* 需求描述：在整个 TUI 界面上方增加一行专属展示区（Header）。
* 展示内容：动态展示当前处于选中状态的会话详细信息，包括 `SessionId`、`Time` 以及相关的 `Project` 工作路径。

### 1.2 鼠标交互支持 (Mouse Support)
* 需求描述：CLI 界面需要启用终端的鼠标捕获 (Mouse Capture) 功能。
* 具体行为：用户可以直接通过鼠标左键点击来选中（focus/highlight）左侧和右侧的任意一个面板（Panel）或条目。此功能也将作为后续更多交互拓展（如悬停提示）的基础。

### 1.3 分栏布局切分与大小调整 (Window Management & Resizing)
* **快速切分布局**：支持动态切换双栏窗口的排列模式。
  * `Ctrl+Shift+H`：应用水平方向分割布局（左/右分栏）。
  * `Ctrl+Shift+V`：应用垂直方向分割布局（上/下分栏）。
* **缩放控制逻辑**：
  1. 系统提供合理的默认 Size 占比分布。
  2. 支持选中目标面板模式：支持上述鼠标点击方式选中面板；同时也支持通过预设快捷键（例如 `Ctrl+Shift+方向左键` / `Ctrl+Shift+方向右键`）在面板间切换并将焦点锁定以供后续调整。
  3. 当某面板被激活选中后，允许通过特定的快捷键执行面积的放大或缩小 (Resize) 操作。

## 2. 核心功能新增 (Functional Additions)

### 2.1 会话恢复功能 (Resume Session)
为避免仅仅查看的单一工具属性，加入快捷调用 Codex 进行对话续录的能力。
* **入口**：在选中任意历史 Session 后，按下 `Enter` 键即可触发本功能。
* **业务逻辑细节**：
  1. **定位项目上下文**：因 `codex` 的特性限制，工具首先需要自动检测该历史文件的元数据 `cwd`，执行类似 `cd` 的操作切换后台进程的工作目录至该项目路径。
  2. **调起命令**：在新目录上下文中立刻拉起命令行子进程：`codex resume <SESSION_ID>`。
  3. **前台终端接管与状态维护**：被拉起的 Codex 进程应该直接接管当前终端的 `Stdin` 与 `Stdout`。此时原有的 TUI 会话工具的渲染界面需暂停更新（进入挂起状态）。
  4. **返场恢复**：当上层 Codex 程序退出后，本工具能够安全取回控制权，重绘终端视图而不产生输入环路的乱码。
