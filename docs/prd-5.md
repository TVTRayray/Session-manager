# 补充需求文档 (PRD-5) - 会话列表视图重构与分组增强

本文档旨在对左侧会话列表的展示形式进行彻底的“人类可读性”重构，废弃原本无意义的 UUID 平铺展示，引入类似主流 IDE 的树状分组视图，并在视觉上深度适配 `oxker` 的极客设计美学。

## 1. 核心组件升级
* **依赖引入**：鉴于 `ratatui` 原生不支持复杂的树形结构，为降低手写状态机的维护成本，规定引入第三方生态库 `tui-tree-widget`。
* **状态托管**：折叠/展开等 UI 层面的 Node 状态，必须统一交由 `TreeState` 维护，禁止业务层自行管理复杂的 UI 显示状态。

## 2. 数据层建模与映射规则 (Data Modeling)

### 2.1 会话摘要提取
废除直接在界面呈现 Session ID 字符串的做法。
* 解析 `.jsonl` 时，检索到第一条 `role: "user"` 的非系统级对话内容，将其作为该会话的 **摘要 (Summary)**。
* 在内存中为每条历史记录建立结构：包含 `摘要`、`时间`、`项目(cwd)` 以及底层关联的 `UUID`。

### 2.2 双轨分组视图 (Group By Views)
系统必须支持在内存中对全量会话数据进行实时的二次聚类，支持以下两种视图模式的无缝切换：
1. **按时间分组 (By Time)**：划分为 `今天`、`昨天`、`过去 7 天` 等高层概念（或简单按日期字符串分组）。
2. **按项目分组 (By Project)**：根据会话元数据中的项目路径 (`cwd`) 名作为根节点分组。

## 3. UI 交互与 Oxker 风格适配 (Aesthetics & UX)

### 3.1 左侧面板树状渲染规范
要求布局紧凑严谨，拒绝繁琐冗长：
* **分组节点 (Group Header)**：
  * 使用全宽、加粗 (Bold) 或 Cyan/Magenta 等高亮配色以体现层级（如：`▼ openagents (共2条会话)`）。
  * 取消杂乱的边框修饰，依赖颜色和留白进行视觉隔离。
* **会话子节点 (Session Leaf)**：
  * 从 Header 向下缩进 2-4 个字符宽度，时间戳要求竖向对其。
  * 渲染格式参考：`[05-06 17:31] 帮我看看当前我在使用...`（摘要超出部分必须执行 `...` 截断）。

### 3.2 右侧详情面板的“空窗期”处理
* 当用户在左侧光标仅聚焦在**分组节点 (Header)**，而非具体会话时，右侧详情面板不应展示上一条对话（避免错觉）也不应黑屏。
* **预期展示**：在右侧面板渲染一张当前分组的数据统计卡片。例如："Project: openagents | Total Sessions: 2 | Last Active: 05-06 17:31"。

## 4. 快捷键定义网络 (Hotkey Mapping)

必须对焦点处于树状图时的快捷键进行严格的功能拆解，避免键位冲突。

* **`g` (Group By Toggle)**：
  * 触发行为：在 `By Time` 和 `By Project` 之间实时切换左侧列表的挂载模式。
  * 状态体现：左侧面板的 Title 需要更新，例如 `[Sessions (By Project)]`。
* **`<-` / `->` 或 `Space` 键 (展开与折叠)**：
  * 如果当前光标悬停在 `Group Header` 节点，按下 `Space` 或方向键可展开或收起该节点。
* **`Enter` 键 (执行与恢复)**：
  * **行为约束**：`Enter` 键只在光标处于叶子节点（具体的 Session 记录）时方可生效。
  * 触发后进入 PRD-3 / PRD-4 定义的 `codex resume` 或 `claude` 终端接管流程。如果停在 Header 层按 Enter，执行静默或视为 `Space` 的展开行为，绝不抛出越界异常。
* **`d` / `Delete` 键 (危险操作)**：
  * 在叶子节点：删除对应会话文件。
  * 在 Header 节点：弹出高危确认窗（"是否永久删除该项目/时间段下的所有 N 条历史会话记录？(y/N)"）。
