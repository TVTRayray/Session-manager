# Spec 15: Bottom Bar Refactor & Help Modal

## 基本信息

- Spec ID：`15-bottom-bar-refactor-and-help-modal`
- 标题：底部状态栏精简与帮助浮层
- 当前状态：`passed`
- 关联阶段：阶段五
- 当前责任角色：`Master`

## 背景与目标

当前底部状态栏显示所有快捷键提示，总长度超过 150 字符，在标准 80-100 列终端下必定发生截断，严重影响可用性。

本 slice 的目标是：**将底部状态栏精简为仅显示高频核心操作的极简文案，并引入 `?` 键触发的帮助浮层展示完整快捷键列表**。

## 用户故事 / 成功标准

- 作为：Codex/Claude 用户
- 我希望：底部状态栏简洁不截断，并能随时按 `?` 查看完整快捷键帮助
- 从而：在小窗口下也能清晰看到核心操作提示，需要时可查阅完整文档

成功标准：

- [ ] 底部常驻栏文案精简为：`Tab:Engine | j/k:Nav | Enter:Toggle/Resume | d:Del | ?:Help | q:Quit`
- [ ] 常驻栏长度控制在 70 字符以内，在 80 列终端下不截断
- [ ] 按 `?` 键触发帮助浮层（Popup Modal）
- [ ] 浮层居中显示，包含所有未在常驻栏展示的高阶快捷键：
  - `Ctrl+Alt+H/V` : 切换面板布局
  - `Ctrl+Alt+←/→` : 切换面板焦点
  - `Ctrl+Alt+=/-` : 动态调整大小
  - 其他已有快捷键
- [ ] 按任意键或 `Esc` 关闭帮助浮层

## 非目标

- 不修改快捷键绑定本身
- 不改变现有功能的触发逻辑
- 不引入可配置的快捷键映射

## 接口与契约

- 输入：
  - 用户按键事件（`?` 或 `h`）
  - 当前已注册的快捷键列表
- 输出：
  - 渲染后的帮助浮层（居中的 Rect + 样式化文本）
- IPC / API：
  - 复用 `ratatui` 的 `Clear` widget 清除浮层背景
  - 复用 `Block` widget 实现浮层边框
- 异常 / 错误返回：
  - 终端高度不足（< 20 行）时，浮层内容可滚动或截断提示

## 数据与状态变化

- 新增状态：
  - `show_help_modal: bool` —— 标记帮助浮层是否可见
- 变更状态：无
- 持久化影响：无

## 边界与失败场景

- 浮层触发时，底层列表和详情面板的刷新需暂停，避免浮层被覆盖
- 多次快速按 `?` 不应导致状态翻转闪烁
- 浮层关闭后，焦点需恢复到触发前的状态（左侧列表或右侧详情）

## 实施要求

1. 精简底部状态栏渲染逻辑，使用硬编码的短文案
2. 新增 `show_help_modal` 状态字段
3. 在主事件循环中拦截 `?` 按键，切换浮层状态
4. 实现浮层渲染逻辑：计算居中位置，绘制边框和内容
5. 在浮层可见时，拦截所有按键（除 `Esc`）为关闭操作

  2. `help_modal_renders_all_shortcuts` 测试不含实际渲染断言，若浮层文案被意外修改，现有测试不会失败。
