# Spec 01 - Session Catalog MVP

## 基本信息

- Spec ID：`01-session-catalog-mvp`
- 标题：会话目录与双栏 TUI 骨架
- 当前状态：`qa-passed`
- 关联阶段：`阶段一：Readable MVP`
- 当前责任角色：`QA`

## 背景与目标

- 当前项目尚无正式的 vertical slice 文档和实现入口，首先需要一个最小可浏览产品，把 `~/.codex/sessions` 中的会话目录稳定映射到双栏 TUI。
- 本 slice 的目标是交付“左侧列表可浏览、右侧详情有响应、底部状态栏可反馈”的最小闭环，为后续清洗渲染和删除能力提供统一的 UI 状态模型。
- 本 slice 的实现语言已固定为 `Rust`，默认采用 `ratatui` + `crossterm` 作为终端 UI 基础设施。
- 当前已发现的线上问题是：实现只扫描 `~/.codex/sessions` 的直接子项，没有递归读取任意层级子目录中的 `.jsonl` 会话文件，导致按年月日等目录结构分层存放的真实会话无法出现在左侧列表。

## 用户故事 / 成功标准

- 作为：Codex CLI 的重度用户
- 我希望：启动工具后立即看到按时间排序的会话列表，并能在双栏界面中切换焦点查看对应详情
- 从而：快速定位我要回顾的会话，不必手工打开原始 JSONL 文件

成功标准：

- [x] 启动后递归读取 `~/.codex/sessions` 下任意层级子目录中的 `.jsonl` 会话文件，并统一按时间倒序展示。
- [x] 左侧列表至少展示 `session_id`、`display_time`、`cwd_tail` 三类信息。
- [x] 用户使用方向键或 `j/k` 时，左侧选中项改变，右侧同步更新为对应详情占位或基础元信息。
- [x] 右侧更新不能阻塞主界面输入，界面必须有可见的加载态或占位态。
- [x] 底部状态栏持续展示快捷键提示和最近一次全局错误或提示信息。
- [x] 根目录直系 `.jsonl` 与多层嵌套目录中的 `.jsonl` 都必须进入同一会话列表；非 `.jsonl` 文件与空子目录必须被忽略。

## 非目标

- 不要求完成对话流清洗和 Markdown/代码块高亮。
- 不包含删除操作、确认弹窗或文件系统写入。
- 不要求满足 100MB+ 文件的极限性能验收，但状态模型必须支持后续异步加载演进。

## 接口与契约

- 输入：
  - 固定目录：`~/.codex/sessions`
  - 文件候选：`~/.codex/sessions` 目录树下任意层级、扩展名为 `.jsonl` 的文件
- 输出：
  - `SessionListItem` 视图模型，至少包含：
    - `session_id`
    - `display_time`
    - `cwd_tail`
    - `abs_path`
    - `is_loadable`
  - `SessionDetailState`，至少包含：
    - `idle`
    - `loading`
    - `ready_stub`
    - `error`
- IPC / API：
  - 仅限本地文件系统读取，不引入网络或外部服务。
  - 会话列表加载与右侧详情触发必须通过可替换的读取接口封装，方便后续单元测试和大文件策略接入。
  - 目录扫描接口必须支持递归遍历根目录下的任意层级子目录，而不是只读取一级目录项。
- 异常 / 错误返回：
  - 目录不存在、目录无权限、文件元数据读取失败时，错误进入状态栏，不允许直接终止程序。
  - 非 `.jsonl` 文件应被静默跳过，不写入错误列表。
  - 对递归扫描过程中发现的符号链接、规范化后越界路径和不可读子目录，必须局部隔离并记录 warning，不得中断其他合法路径的扫描。

## 数据与状态变化

- 新增状态：
  - `session_list`
  - `selected_index`
  - `detail_state`
  - `status_message`
- 变更状态：
  - 启动后初始化列表与默认选中项。
  - 选中项变化时触发右侧详情状态从 `idle/ready_stub` 转到 `loading` 再到 `ready_stub/error`。
- 持久化影响：
  - 无；本 slice 只读，不修改任何业务文件。

## 边界与失败场景

- 空目录：左侧展示空态，右侧显示“无会话”占位，不得 panic。
- 目录中混入非 `.jsonl` 文件：静默跳过，不污染列表。
- 空子目录：静默跳过，不污染列表，不产生伪错误。
- 多层嵌套目录：任意层级内的 `.jsonl` 文件都必须被发现并进入列表。
- 会话文件存在但元数据不完整：允许降级展示文件名派生信息。
- 指向根目录外文件的符号链接：必须拒绝，不纳入列表，并记录 warning。
- 递归过程中遇到无权限目录或读取失败的条目：只影响局部路径，其余合法路径继续扫描。
- 列表为空时用户继续按导航键：输入应无副作用，不得越界。

## 实施要求

- 必须保留双栏布局和底部状态栏，不允许以单页日志查看器替代。
- 左侧列表项字段命名和状态命名必须稳定，供后续 spec 直接复用。
- 详情区域即便只展示占位，也必须经过异步状态流转，不能把未来的大文件读取写死为同步调用。
- 文件系统访问范围必须限制在 `~/.codex/sessions` 目录。
- Rust 实现中不得使用会绕过错误处理的 `unwrap()` / `expect()` 来处理目录扫描、元数据读取和选中态切换等运行时路径。
- TUI 事件循环和状态更新应围绕 `ratatui` 的渲染模型设计，避免后续 Coder 重新定义 UI 基础设施。
- 递归扫描的目标是支持 `~/.codex/sessions` 下任意层级子目录中的 `.jsonl` 文件，不限定为固定年月日层数。
- 不允许为了支持递归遍历而放松现有根目录边界校验；所有候选路径在入列表前都必须通过规范化后归属校验。
